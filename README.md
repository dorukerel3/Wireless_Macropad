Advanced Wireless MacroPad System (ESP32-S3)
This repository contains the firmware and core system architecture for a dual-node wireless macro keyboard system. Engineered around the ESP32 microcontroller ecosystem, this project was developed to resolve the latency, memory degradation, and connectivity limitations frequently encountered in open-source human interface devices. The complete system comprises a primary transmitter node, which acts as the macro pad interface, and a dedicated receiver node functioning as a native USB HID gateway. The two devices communicate seamlessly via the low-latency ESP-NOW protocol.

Transmitter Node Architecture and Memory Decisions
The primary focus during the development of the transmitter firmware was long-term memory stability and strict power management. To ensure continuous operation without risk of kernel panics or performance degradation over extended uptimes, dynamic string allocations were entirely avoided in the core codebase. Instead, the firmware handles all data processing through static multidimensional character arrays and reentrant parsing functions such as strtok_r. This deliberate architectural choice prevents heap fragmentation and ensures a predictable memory footprint.

Configuration management is handled asynchronously through an embedded captive web portal powered by the ESPAsyncWebServer library. The underlying frontend assets, including HTML, CSS, and JavaScript, are stored in continuous flash memory using PROGMEM attributes. Operating the web server on a separate execution context guarantees that users can modify macro configurations on the fly without introducing any blocking delays or jitter into the high-priority physical hardware interrupt loop. The non-volatile storage partitioning is scaled to securely maintain up to 10 independent hardware layers, representing 160 macro slots, with instantaneous state transitions that do not require an HTTP reload.

Radio frequency state control is managed via an autonomous state machine designed to minimize wireless interference and reduce overall power consumption. The device prioritizes authentication with a local station network and automatically deactivates its internal Soft-AP once a secure connection is established. In the event of a signal drop or credential mismatch, the state machine restores the access point and captive portal instantly, ensuring the configuration interface remains reachable. The node also includes a secure multipart upload handler to support Over-The-Air firmware updates directly through the asynchronous web server.

Receiver Node and HID Execution Logic
The receiver node acts as a dedicated hardware bridge between the proprietary 2.4GHz ESP-NOW wireless packets and the host operating system. The receiver listens exclusively for specific struct_message payloads matching the transmitter's unique hardware MAC address. Upon receiving a valid packet, the device leverages the native hardware USB capabilities of the ESP32 core to translate the data into standard USB HID events with near-zero latency. This setup allows the system to execute complex keystrokes, precise mouse actions, and consumer control sequences natively across different operating systems without requiring any specialized software on the host computer.

Repository and Layout Structure
The repository is organized logically to separate the individual development domains of the dual-node ecosystem. The firmware/transmitter/ directory contains the monolithic C++ source code for the macro pad, including the embedded web interface and the wireless transmission logic. The firmware/receiver/ directory houses the gateway logic and the native USB HID translation routines. All physical design files, including circuit schematics, PCB Gerber layouts, and 3D-printable enclosure models, are located within the hardware/ directory.

Deployment and Compilation Protocols
Deploying this system requires an ESP32-S3 development board with native hardware USB CDC support and up to 16 physical mechanical switches routed to GPIO pins 1 through 16 using internal pull-up resistors. Compilation is fully compatible with the Arduino IDE using the ESP32 Core version 3.x.x or higher. Before compiling the transmitter code, the external AsyncTCP and ESPAsyncWebServer dependencies must be added to the build environment from source.

When configuring the compilation environment in the IDE, the target board must be set to ESP32S3 Dev Module, and the USB Mode must be explicitly configured as Hardware CDC and JTAG to enable the required native hardware features. Due to the substantial non-volatile storage matrix required to persist 160 independent macro sequences alongside the wireless update structures, the partition scheme must be set to 8MB Flash or Huge APP. Following a successful flash, establishing the secure link requires reading the receiver's hardware MAC address and inputting it into the transmitter's web configuration panel to register the ESP-NOW peer connection.
